# Live Databento Feed for the Production Book — Feasibility & Transition Plan

**Date:** 2026-07-11
**Goal:** transition the 5 production strategies (gfi, glx, glf, lstb, lgpr/v1-ES) to live Databento data so the live feed matches backtest data identically.
**Verdict: FEASIBLE.** ~$400/mo flat (two Standard plans) + ~$50–65/mo historical usage. The hard part is not the feed — it's that true parity **exposes one remaining lookahead in our own backtest data** (stats-variant GEX) that must be fixed and re-validated first.

## 3. Live API technical facts

- Raw TCP + DBN binary; gateway `{dataset}.lsg.databento.com:13000`; CRAM auth with API key. One dataset per session; Standard = 10 concurrent sessions/dataset (we need ~3 total).
- **ohlcv-1s streams live** (bar arrives ~1s after close). OPRA live = cmbp-1 / tcbbo / **cbbo-1s / cbbo-1m** / statistics / definition. Statistics stream live → day-D OI arrives premarket (~06:00 ET) on the feed.
- **Parent symbology live**: `NQ.FUT` (outrights + calendar spreads — exactly matches our raw CSVs), `QQQ.OPT`/`SPY.OPT` (server-side chain filter, no OPRA firehose; cbbo-1m ≤1 msg/min/instrument).
- **Intraday replay**: `start` up to 24h back, seamless replay→real-time in one session. This is both the reconnect gap-fill (re-subscribe with start=last-seen-ts) and the warm-start after restarts.
- **No Node.js client** (official: Python/Rust/C++; JS is an open roadmap request). Integration = **Python sidecar** running the official Live client, republishing to Redis — matches the data-service architecture. Python client has reconnect_policy, callbacks, symbology_map upkeep, heartbeat timeout detection.
- Maintenance window: CME Sat 02:00 CT. Slow-reader default = `skip` (returns to real-time) for stateless schemas since 2026-04.

## 4. Historical/acquisition API facts

- **Strict 24h embargo — WRONG as of ~Sep 2025.** With a live plan for the dataset, historical is available intraday: **GLBX delayed 20 min, OPRA delayed 1 hour, XNAS delayed 10 min** (without a plan: 8h / next-day 09:30 / next-day 00:00). Live replay covers the last 24h, so live + historical tile with no gap.
- Freshness gating: `metadata.get_dataset_range` (latest entitled ts) + `metadata.get_dataset_condition` per date (`available` / `degraded` / `pending` / `missing`; weekends = missing). Ingest only `available`; re-check `last_modified_date` for restatements.
- Cost preflight: `metadata.get_cost` / `get_billable_size` / `list_unit_prices` — exact dry-run of any pull; get_cost respects plan discounts.
- Streaming `get_range` (<5 GB, re-bills on repeat) vs `batch.submit_job` (>5 GB, free re-download 30 days, daily-ETL recommended). 2,000 symbols/request. Worked incremental-puller example in the research transcript (condition-gated, high-water-mark state, cost ceiling abort).

## 5. Parity blockers found in OUR data (must fix before "identical" means anything)

1. **CONFIRMED & QUANTIFIED (2026-07-11, see `VERIFICATION-stats-gex-lookahead.md`) — stats-variant GEX has an intraday lookahead (affects gfi + lgpr golds).** stat_type 11 close prices publish 16:00–17:00 ET (100% of records, 5 sampled days across 2023–2026, both ts_event and ts_recv); OI publishes premarket. Counterfactual A/B (10 days, 580 snapshots, same code/spots, day-D vs prior-day closes): **regime label flips on 22% of snapshots, put wall moves 58%, call wall 47%, top-5 support overlap only 0.55, gamma_flip moves 200–925 pts median on 4/10 days.** Not cosmetic — gfi trades flip crossings/regime; lgpr classifies path-blocked from these levels. The 2026-04/05 campaign fixed the cbbo dirs + IV CSV and *added* the causal `--iv-source cbbo` option, but the stats dirs were never regenerated. Two more findings from verification: **(B)** `load_ohlcv_for_date` spot selection is row-order-sensitive on the multi-contract raw CSV → shipped stats GEX is no longer exactly reproducible (constant-ratio level scaling); **(C)** `data/gex/es` never received the 2026-05-06 as-of relabel → lgpr's ES gate also carries the ~14-min spot-label lookahead.
   **Fix:** regen `gex/nq` with `--iv-source cbbo`; `gex/es` needs relabel + cbbo IV (SPY cbbo backfill ~$350, or interim prior-day-close causal variant); fix the spot loader's contract selection in the same pass; re-run gfi v2 + lgpr v1/v1-ES golds; re-baseline the FCFS book. **This is the long pole of the whole transition.**
2. **cbbo variant (glx, glf) is already causal** post the 2026-04-30/05-06 fixes (ts_recv bucketing, as-of labels). Reminder landmine: committed `generate-cbbo-gex.js` ≠ committed `nq-cbbo` data — regen breaks GLX v3 gold; reconcile before any regen.
3. **Equities spot source seam:** backtest QQQ/SPY 1m = XNAS.ITCH; live would be EQUS.MINI/IEX (or Schwab fallback). Consolidated vs Nasdaq-only 1m closes differ by ~cents. Verify with a bar-diff over an overlap window; if material, single-source both sides (EQUS both directions going forward, document the seam date).
4. **Primary-contract selection:** backtest `filterPrimaryContract()` picks the per-hour highest-volume contract retrospectively; live pins the front month via `*_CONTRACT` env vars. Divergence risk is confined to roll days. Mitigation: the connector computes per-hour volume by contract from the parent subscription and alerts when the non-pinned contract wins an hour.
5. **Candle emission semantics:** Databento 1s bars → connector assembles 1m with the exact `import-databento-data.js` bucketing (Map-of-open-buckets; the (ts,symbol) ordering bug is already fixed) and publishes `candle.close` at the minute boundary (~1s latency vs Schwab's ~2s single-bar delivery — strictly better, and removes the TV-vs-Schwab semantics split).

## 6. Target architecture

**New: `databento-connector` (Python sidecar, deployed with data-service; single replica like the Schwab streamer):**
- Session 1 — GLBX.MDP3, `NQ.FUT`+`ES.FUT` parent, ohlcv-1s: aggregate to 1m with import-script bucketing; publish `price.update` (1s) + `candle.close` (1m) per contract; track SymbolMappingMsg; per-hour volume-by-contract telemetry (roll guard).
- Session 2 — OPRA.PILLAR, `QQQ.OPT`+`SPY.OPT` parent, cbbo-1m + statistics + definition: accumulate day stats (OI premarket) + running quote map with `load_cbbo_buckets` forward-fill semantics; **every 15 min run the same GEX code** and publish `gex.levels`. Refactor `generate-intraday-gex.py` into a shared library (`gex_lib.py`) imported by both the offline generator and the connector — same-code-both-sides is what makes byte-parity possible.
- Session 3 — EQUS.MINI (or IEX), QQQ+SPY ohlcv-1m: ETF spot for translation + GEX inputs (pending plan-cost confirmation; Schwab fallback).
- Reconnect: `reconnect_policy` + re-subscribe with `start`=last-seen (24h replay backfills the gap). Publish staleness alerts (fail-LOUD — the June 9-10 GEX outage was fail-silent).

**New: nightly acquisition cron** (per the worked example; condition-gated, cost-preflight, high-water-mark state): pulls T-1 GLBX NQ+ES 1s/1m, OPRA QQQ+SPY cbbo-1m/statistics/definition, equities 1m → appends the backtest CSVs, regenerates GEX dirs, runs an **automated live-vs-backtest diff** (yesterday's live-published candles/GEX vs the freshly regenerated backtest versions). This permanently kills the "data walls at date X" problem AND gives a standing parity audit — the thing the June-2026 audit lacked.
